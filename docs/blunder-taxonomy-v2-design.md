# 悪手分類・難易度推定の拡張設計（v2検討）

作成: 2026-07-17。現行実装（classifier v1・係数表v1）を前提にした将来拡張の設計メモ。

## 現状（v1）の構造

- **分類**: ルールベース6カテゴリ（BlunderClassifier.kt）。PV統計（駒割差・王手数・
  詰みの有無）から `詰み見逃し／頓死／駒損（即取り）／駒損（タクティクス）／
  玉の危険（寄せ）／位置的・その他` に分岐
- **難易度（相応判定）**: カテゴリ×レート帯の実測見逃し率
  （coefficients_hao_v1: lishogi 11,547局・1,060,887手・65,638悪手。
  詰みは手数バケット別 mate_miss）
- **出題**: 見逃し率由来の priority で「その棋力帯が直すべき悪手」を優先

### v1の限界

1. **「位置的・その他」が最大バケット**（1600-1899帯で32.1/1000手＝全カテゴリ中最大）。
   玉石混交で、教材としての説明力（note・problem名）が最も弱い
2. **難易度の粒度がカテゴリ単位**。同じ「駒損（タクティクス）」でも
   3手の両取りと9手の攻防は学習価値も正答率も別物
3. **必至・詰めろの厳密判定なし**（df-pn未実装。「詰み手数表記は詰み筋とは
   限らない」の注記で回避中）
4. モチーフ（両取り・素抜き等）は problem 文言レベルにしか存在せず、
   集計・出題制御の軸になっていない

## 提案: 3層アーキテクチャ

### 層1: モチーフタグ（種別の高解像度化）

単一カテゴリ→**複数タグ集合**へ。lichess puzzle themes の先例
（lichess-puzzler の cook.py が PV から60超のテーマを機械抽出）の将棋版。
すべて既存の解析出力（bestPv/punishPv＋盤面）から機械的に抽出できるものだけで構成する:

| タグ候補 | 抽出方法 |
|---|---|
| 両取り（fork） | punish PV 中の1手が2枚以上の駒に当たる |
| 素抜き・田楽刺し | 利きの直線上に価値駒が並ぶ形の検出 |
| 垂れ歩・叩き | 打ち歩＋次手の成り/取りの形 |
| 捨て駒 | 直後に取られるが評価が維持される手 |
| 中合い・移動合い | 王手に対する合駒の種別 |
| 成らず・不成 | 手の表記から直接 |
| 局面フェーズ | 序盤/中盤/終盤（手数・駒交換数・玉の接近度・持ち駒量） |
| 攻守方向 | 悪手の悪化が自玉起点か敵玉起点か（PVの王手方向から） |

- **DB**: blunder レコードに `tags`（JSON配列）を1列追加するだけ。
  classifier_version は解析結果に記録済みなので、v1/v2並走と
  既存棋譜の再分類バッチが最初から可能な構造になっている
- 既存6カテゴリとv1係数表は後方互換のため当面併存（タグは付加情報として導入）

### 層2: 局面難易度の機械指標

- **(a) depth-to-find**: nodes=1k/4k/16k/64k/256k の階段探索で、最善手が
  首位に浮上する最小ノード数＝「浅い読みで見える手か」の機械指標。
  **不変条件（400kノード=研究の物差し）とは独立の追加計測**なので較正を壊さない。
  計測はドリル出題候補だけに限定すれば実機負荷も許容範囲
- **(b) 唯一手度**: MultiPV=2 は既にあるので best/2nd の評価差を記録するだけ。
  ただし「唯一手=難しい」ではないため (a)(c) と組み合わせて使う
- **(c) 人間的見落とし形**: 成らず・後退する手・離れ駒への打ち込み・中合い等、
  人間が候補から外しやすい手形のタグ（層1と共通の抽出器）

### 層3: 実測難易度（本命・データ駆動）

- ドリル正答履歴は既にDB保存されている。**Supabaseデータ提供が開通した（2026-07-17）**
  ので、ユーザー横断の正答データが貯まる基盤は完成済み
- 将来形: 問題ごとに **Glicko系レートを付与**（lichess puzzle rating 方式）。
  ユーザー側レートは推定棋力（既存）を初期値に、ドリルの正誤で相互更新
- コールドスタート: 層1タグ×層2指標から初期難易度を回帰で予測し、
  実測が貯まった問題から実測値に置き換える

## 段階導入計画

| Phase | 内容 | アプリ変更 |
|---|---|---|
| A | research側でlishogi 65,638悪手にモチーフ抽出器を実行し、タグ×レート帯の見逃し率を計測。**「位置的・その他」をどれだけ解体できるか**を評価指標に、v2係数表を設計 | なし |
| B | `tags`列＋classifier v2＋既存棋譜の再分類バッチ | スキーマ+1列 |
| C | depth-to-find計測（ドリル候補のみ）。noteを「あなたの棋力帯の◯◯（タグ）の見逃し率」へ高解像度化。出題優先度を係数表v2へ切替 | 解析パイプライン追加 |
| D | ドリル正答データによる問題レート導入 | 出題ロジック |

Phase A はアプリに一切触らず研究だけで価値が測れる（v2タグの弁別力が
出なければ止められる）ので、着手点として最適。

## 不変条件との整合

- `go nodes 400000・Threads=1・MultiPV=2・FV_SCALE=20` は不変。層2(a)は別パスの追加計測
- v1係数表・6カテゴリ・悪手定義バージョンの記録機構はそのまま。v2は
  バージョン併記で導入し、切替は Phase C の係数表v2とセットで行う

## Phase A 結果（2026-07-17 実施済み・成立）

research側で63,771悪手に対して検証済み（research/docs/motif-extraction-phase-a.md）:

- 「位置的・その他」の**73.6%**が実質タグに分解できた
- 全タグの1000手あたり率が帯に対して**単調減少**（弁別比×3.0〜×6.5）。
  最強の弁別子は best_retreat（後退する最善の見逃し・×6.5）
- best_nonpromo のみ弁別が弱く（×1.3）、相応判定でなく難易度素材（層2）へ回す

→ Phase B（tags列＋classifier v2）へ進む判断材料が揃った。
